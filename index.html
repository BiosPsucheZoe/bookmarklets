Making Accessibility Visible

We created these bookmarklets in order to:

  • Make otherwise hidden accessibility features on web pages (like ARIA landmarks, roles, labels and descriptions and alternative text for non-text objects) more visible
  • Show whether HTML markup (like lists, headings, label elements, and appropriate landmark roles) is used with proper semantics for accessibility
  • Flag elements or regions on web pages where information that could improve accessibility (like elements not contained in landmarks, image alt text or long descriptions, list accessible names or descriptions, or form control labels) is missing

Usage

To get started using the bookmarklets, follow the installation instructions.

Our bookmarklets are based on WCAG 2.0, ARIA 1.0, and HTML5 requirements. Their purpose is to help web developers and designers see the "big picture" regarding web accessibility by providing information that relates the design of accessible web resources. They are not intended to provide specific compliance results or recommendations.

Contributors

These bookmarklets were originally created by Disability Resources and Educational Services (DRES) at the University of Illinois at Urbana-Champaign and Pixo. The bookmarklets and this web page are open source in GitHub, and anyone can contribute.
